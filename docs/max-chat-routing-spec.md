# Спецификация: маршрутизация MAX в Telegram-топики

## Зачем это нужно

Старая модель требовала вручную привязывать каждый чат MAX к отдельной Telegram-группе или общей группе.

Новая модель проще для ежедневной работы:

- владелец один раз выбирает одну Telegram-супергруппу с включенными темами;
- бот автоматически создает отдельный топик для каждого чата, группы или канала MAX;
- рабочая переписка не смешивается, но все остается в одном Telegram-пространстве;
- историю топиков владелец удаляет сам, если она больше не нужна.

## Что изменится для пользователя

После доработки бот работает так:

- в личке с владельцем: статусы, ошибки, выбор главного Telegram-форума и управление отслеживанием;
- в выбранной Telegram-супергруппе: отдельные топики для чатов MAX;
- в каждом топике: сообщения из одного источника MAX и ответы владельца обратно в этот же источник MAX.

В результате:

- больше не нужно выбирать Telegram-группу для каждого MAX-чата;
- новый MAX-чат сразу получает свой топик;
- название топика берется из названия MAX-группы/канала, а для личного чата - из имени собеседника;
- если MAX-чат переименован, бот переименовывает соответствующий Telegram-топик;
- если чат отключен или владелец вышел из MAX-группы, существующий топик остается как история.

## Основные роли

- Владелец бота: один Telegram-пользователь, чей `user_id` хранится в `.env` как `TG_ADMIN_ID`.
- Рабочий Telegram-форум: одна супергруппа с включенными темами, выбранная владельцем через кнопку в личке с ботом.
- MAX-чат: источник входящих сообщений из MAX.
- Топик MAX-чата: отдельная тема внутри рабочего Telegram-форума.

## Целевое поведение

### 1. Первичная настройка рабочего форума

Владелец открывает личку с ботом и нажимает кнопку выбора рабочего форума.

Бот просит выбрать Telegram-супергруппу, в которой:

- включены темы;
- бот добавлен участником;
- бот является администратором;
- у бота есть право управлять темами.

После выбора бот сохраняет `chat_id` форума локально. Если прав не хватает, бот пишет владельцу понятное сообщение и не начинает автоматическое создание топиков.

### 2. Автоматическое создание топиков

Когда бот видит MAX-чат:

1. Он создает или обновляет локальную запись MAX-чата.
2. Если рабочий форум выбран и доступен, он создает в нем топик.
3. В локальном JSON сохраняется `message_thread_id`.
4. Все дальнейшие сообщения этого MAX-чата отправляются в сохраненный топик.

Если топик создать нельзя из-за прав, настроек форума или ошибки Telegram, сообщения не теряются. Они ставятся в локальную очередь, а владелец получает уведомление в личку.

### 3. Названия топиков

Название топика должно быть понятным владельцу:

- для MAX-группы или канала используется название группы/канала;
- для личного MAX-чата используется имя собеседника;
- если название слишком длинное, оно обрезается до лимита Telegram;
- если MAX-чат переименован, бот переименовывает уже созданный топик.

### 4. Пересылка сообщений из MAX

Правила маршрутизации:

- если чат MAX активен и топик создан, сообщение идет в этот топик;
- если чат MAX активен, но топик еще не создан, сообщение ставится в очередь;
- если чат MAX помечен как `Не отслеживать`, сообщения игнорируются;
- технические сообщения всегда идут только владельцу в личку.

### 5. Ответы из Telegram обратно в MAX

Ответы работают двумя способами.

Обычное сообщение владельца в топике:

- любое текстовое сообщение от владельца в топике отправляется в соответствующий MAX-чат обычным сообщением;
- сообщение владельца в Telegram не удаляется;
- исходное сообщение из MAX в Telegram не редактируется и не дополняется блоком ответа.

Reply на пересланное сообщение из MAX:

- если владелец отвечает в Telegram через reply на сообщение, пришедшее из MAX, бот отправляет ответ в MAX как reply на исходное MAX-сообщение;
- если исходный MAX `message_id` недоступен, бот отправляет обычное сообщение или сообщает владельцу о невозможности reply;
- отдельной кнопки под сообщением нет: для ответа на конкретное сообщение используется стандартный Telegram reply.

Отправлять сообщения обратно в MAX может только владелец. Сообщения других участников топика бот игнорирует.

### 6. Управление чатами

В личке с ботом владелец видит:

- выбранный рабочий Telegram-форум;
- список известных MAX-чатов;
- статус топика для каждого чата;
- кнопку отключения/включения отслеживания.

Выбор отдельной Telegram-группы для MAX-чата больше не используется.

### 7. Отключение или исчезновение MAX-чата

Если владелец отключил чат, вышел из MAX-группы или удалил чат:

- бот перестает пересылать новые сообщения;
- Telegram-топик не удаляется;
- историю владелец чистит вручную при необходимости.

## Что сохраняется локально

Локальное хранилище живет в `data/chat_bindings.json`.

Глобально хранится:

- ID выбранного Telegram-форума;
- название выбранного Telegram-форума;
- признак доступности форума и текст последней ошибки.

Для каждого MAX-чата хранится:

- ID чата MAX;
- понятное название MAX-чата;
- тип MAX-чата;
- статус отслеживания;
- ID Telegram-форума;
- `message_thread_id` топика;
- название топика;
- очередь недоставленных сообщений;
- служебные признаки для уведомлений и повторных попыток.

Старые привязки MAX-чата к отдельным Telegram-группам больше не используются. При обновлении бот должен создать новые топики заново.

## Что исключено из этой итерации

В эту итерацию не включаем:

- автоматическое создание самой Telegram-супергруппы;
- автоматическое включение тем в Telegram-супергруппе;
- автоматическое удаление топиков;
- синхронизацию аватарок;
- несколько владельцев;
- веб-панель или отдельную БД.

## Критерии приемки

1. Владелец выбирает один рабочий Telegram-форум через кнопку в личке с ботом.
2. Бот проверяет, что выбранный чат является форум-супергруппой и что у бота есть право управлять темами.
3. Для каждого активного MAX-чата бот автоматически создает отдельный топик.
4. Сообщения MAX-чата отправляются в его `message_thread_id`.
5. Название топика создается из названия MAX-группы/канала или имени собеседника в личном чате.
6. При переименовании MAX-чата бот переименовывает Telegram-топик.
7. Старые привязки к отдельным Telegram-группам не используются.
8. Любое текстовое сообщение владельца в топике отправляется в соответствующий MAX-чат.
9. Reply владельца на пересланное сообщение MAX отправляется в MAX как reply на исходное сообщение, если исходный MAX `message_id` известен.
10. Сообщения владельца в Telegram не удаляются, а исходные сообщения из MAX не редактируются после ответа.
11. При отключении или исчезновении MAX-чата Telegram-топик остается как история.
