# Спецификация: маршрутизация чатов MAX по Telegram-группам

## Зачем это нужно

Сейчас бот складывает сообщения из всех чатов MAX в один Telegram-чат, указанный в `.env`.

Для живой работы это неудобно:

- переписка из разных чатов смешивается;
- новые чаты приходится разбирать вручную по заголовкам;
- технические сообщения и обычные сообщения приходят в одно место;
- нельзя гибко отключить отдельный чат без правки `.env`.

Цель доработки: сделать так, чтобы чаты MAX можно было разводить по разным Telegram-группам или объединять несколько чатов в одну общую группу, а управление происходило из личного диалога с ботом.

## Что должно измениться для пользователя

После доработки бот будет работать в двух режимах одновременно:

- в личке с владельцем бота: статусы, ошибки, уведомления о новых чатах, настройка привязок;
- в Telegram-группах: только рабочая переписка конкретных MAX-чатов.

В результате:

- один чат MAX можно привязать к своей Telegram-группе или к общей группе вместе с другими чатами MAX;
- новые чаты MAX не теряются и не попадают в общий поток;
- владелец может быстро привязать чат или отключить его без редактирования конфигов;
- отвечать обратно в MAX может только владелец, даже если группа общая.

## Основные роли

- Владелец бота: один Telegram-пользователь, чей `user_id` хранится в `.env`.
- Telegram-группа назначения: группа или супергруппа, которая может быть привязана к одному или нескольким чатам MAX.
- Чат MAX: источник входящих сообщений.

## Целевое поведение

### 1. Личный чат с ботом

Бот пишет владельцу в личку:

- сообщение о подключении MAX;
- сообщение о потере соединения и переподключении;
- ошибки авторизации и доставки;
- уведомления о новых чатах MAX;
- экран управления с кнопкой `Настроить чаты`.

Бот не должен писать технические сообщения в рабочие Telegram-группы.

### 2. Экран `Настроить чаты`

В личке бот показывает список известных чатов MAX.

Для каждого чата доступны действия:

- `Привязать` — выбрать Telegram-группу для пересылки;
- `Не отслеживать` — исключить чат из пересылки;
- `Включить обратно` — вернуть чат из состояния `Не отслеживать`;
- `Сменить группу` — перепривязать чат к другой группе;
- `Отвязать` — убрать текущую привязку и вернуть чат в состояние `Требует настройки`.

В списке рядом с названием чата показывается текущий статус:

- `Привязан`;
- `Ожидает добавления бота`;
- `Не отслеживается`;
- `Не настроен`.

### 3. Привязка чата к Telegram-группе

Пользовательский сценарий такой:

1. Владелец нажимает `Привязать` у нужного чата MAX.
2. Бот отправляет временную кнопку выбора Telegram-группы.
3. Владелец выбирает нужную группу.
4. Бот сохраняет привязку по `chat_id` выбранной Telegram-группы в локальный JSON.
5. Если бот уже состоит в этой группе, пересылка может начаться сразу.
6. Если бот ещё не добавлен в группу, чат получает статус `Ожидает добавления бота`.

### 4. Поведение, если группа выбрана, но бот ещё не добавлен

Если владелец привязал группу, но ещё не добавил в неё бота:

- привязка считается сохранённой;
- бот пишет в личку, что чат ожидает добавления бота в выбранную группу;
- бот не теряет саму настройку;
- после появления доступа к группе бот начинает пересылку в неё.

### 5. Уведомление о новых чатах MAX

Если в MAX появился чат, которого ещё нет в локальной настройке:

- бот пишет владельцу в личку;
- в сообщении показывает название нового чата;
- даёт две кнопки: `Привязать` и `Не отслеживать`.

Если владелец пока ничего не выбрал, чат остаётся в состоянии `Не настроен`.

### 6. Кто может отправлять ответы из Telegram обратно в MAX

Отвечать обратно в MAX может только владелец бота.

Это правило действует везде:

- в личке с ботом;
- в привязанных Telegram-группах.

Если другой участник группы нажимает inline-кнопки или пытается отправить ответ в MAX, бот должен отказать без выполнения действия.

### 7. Пересылка сообщений из MAX

Правила маршрутизации:

- если чат MAX привязан и группа доступна, сообщения идут в эту группу;
- если чат помечен как `Не отслеживать`, сообщения игнорируются;
- если чат `Не настроен`, бот не пересылает его сообщения и напоминает о необходимости решения в личке;
- если чат `Ожидает добавления бота`, бот временно не доставляет сообщения в Telegram-группу и возобновляет доставку после появления доступа.

## Что сохраняется локально

Для каждого чата MAX локально хранится:

- ID чата MAX;
- понятное название чата MAX;
- статус настройки;
- ID Telegram-группы, если она выбрана;
- понятное название Telegram-группы, если оно известно;
- время последнего изменения;
- служебные признаки, нужные для уведомлений и повторной проверки доступа.

Хранилище должно жить в локальном JSON-файле внутри проекта.

## Что исключено из первой итерации

В первую итерацию не включаем:

- автоматическое создание Telegram-групп самим ботом;
- автоматическую установку аватарки Telegram-группы из MAX;
- синхронизацию переименований и аватарок MAX-чата в уже созданной Telegram-группе;
- управление несколькими владельцами;
- веб-панель или отдельную БД.

## Технические решения, которые фиксирую для реализации

### Конфиг

Вместо единого `TG_CHAT_ID` бот будет использовать обязательный `TG_ADMIN_ID`:

- `TG_ADMIN_ID` — Telegram `user_id` владельца бота;
- старый режим с общей группой больше не используется;
- бот должен запускать Telegram polling всегда, а не только при `REPLY_ENABLED=true`, потому что управление теперь постоянно живёт в Telegram.

### Хранилище

Для первой версии достаточно локального JSON, например:

- `data/chat_bindings.json` — привязки и состояния чатов;
- при необходимости отдельный JSON для временной очереди недоставленных сообщений.

### Права доступа

Бот принимает административные действия только от `TG_ADMIN_ID`.

### Reply в MAX

Текущая логика ответа из Telegram в MAX сохраняется, но:

- reply-кнопки появляются в привязанных группах конкретного чата;
- отправка обратно в MAX разрешена только владельцу;
- технические состояния ответа не должны смешиваться между разными группами.

## Зафиксированное правило ожидания доступа к группе

Если Telegram-группа уже выбрана, но бот ещё не добавлен в неё:

- сообщения из MAX не теряются;
- они ставятся в локальную очередь ожидания;
- после появления доступа к группе очередь дозаливается туда по порядку;
- если ожидание затянулось или очередь начинает заметно расти, бот пишет владельцу в личку служебное предупреждение.

Это правило зафиксировано как обязательное для первой версии.

## Критерии приёмки

1. Технические сообщения бот пишет только владельцу в личку.
2. Для каждого отслеживаемого чата MAX можно назначить отдельную Telegram-группу без правки `.env`, либо направить несколько чатов MAX в одну общую группу.
3. Новый чат MAX не попадает молча в общий поток, а требует явного решения владельца.
4. Чат можно пометить как `Не отслеживать` и затем включить обратно из интерфейса бота.
5. Если бот ещё не добавлен в выбранную группу, привязка не теряется.
6. После появления доступа к группе пересылка в неё начинает работать.
7. Ответы обратно в MAX принимает только владелец бота.
