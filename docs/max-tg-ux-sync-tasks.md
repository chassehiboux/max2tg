## Чек-лист

### 1. Убрать префикс имени Telegram из текста, уходящего в MAX

Статус: `done`

- убрать добавление строки `💬 Имя:` для group/supergroup;
- сохранить текущую отправку текста в MAX;
- обновить или добавить тесты для `tg_handler.py`.

### 2-3. Единый блок улучшения UX ответа в Telegram

Статус: `done`

- сохранять ID сообщения-подсказки;
- удалять его после первого ответа пользователя;
- удалять его при `/cancel`;
- сохранять ID исходного телеграм-сообщения при нажатии кнопки;
- после успешной отправки редактировать исходное сообщение;
- добавлять блок:

```text

<текст ответа>
```

- предусмотреть fallback, если сообщение Telegram нельзя редактировать;
- покрыть сценарии тестами.

### 5. Добавить выбор режима ответа: обычное сообщение или reply

Статус: `done`

- определить UX кнопок выбора режима;
- хранить `mode` и `max_message_id` в состоянии ожидания ответа;
- реализовать обычную отправку в чат MAX;
- через `playwright` открыть `web.max.ru` в сохранённом контексте;
- подтвердить формат payload reply в веб-клиенте MAX;
- после подтверждения формата payload реализовать настоящий `reply` в MAX.

## Порядок выполнения

1. Реализовать задачу 1.
2. Отдельно взять задачи 2 и 3 как общий блок улучшения UX Telegram.
3. После этого переходить к задаче 5.

## Что требует отдельной проверки перед реализацией

- точный формат отправки reply на конкретный `message_id` в MAX;
- ограничения Telegram на редактирование некоторых типов сообщений.
