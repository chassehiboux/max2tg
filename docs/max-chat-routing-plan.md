# План: маршрутизация MAX в Telegram-топики

## Цель

Перевести проект с модели ручных привязок MAX-чатов к Telegram-группам на модель одного Telegram-форума, где каждый MAX-чат получает свой топик автоматически.

## Этапы

### Этап 1. Зафиксировать новую модель настройки

Что изменится для пользователя:

- владелец один раз выбирает рабочую Telegram-супергруппу с темами;
- отдельный выбор группы для каждого MAX-чата исчезает;
- старые привязки не используются.

Что сделать технически:

- добавить глобальную настройку рабочего форума в локальное JSON-хранилище;
- хранить `tg_forum_chat_id`, название форума, доступность и последнюю ошибку;
- добавить выбор форума через кнопку `request_chat`;
- запрашивать именно forum supergroup и права на управление топиками.

### Этап 2. Научить бота создавать и поддерживать топики

Что изменится для пользователя:

- новый чат MAX автоматически получает отдельный топик;
- название топика понятно соответствует источнику MAX;
- переименование MAX-чата отражается в Telegram.

Что сделать технически:

- добавить в запись MAX-чата поля `tg_forum_chat_id`, `tg_topic_id`, `tg_topic_name`;
- создать метод, который гарантирует наличие топика для активного MAX-чата;
- использовать `create_forum_topic`;
- использовать `edit_forum_topic` при изменении названия;
- обрезать названия до лимита Telegram.

### Этап 3. Перенаправить доставку в `message_thread_id`

Что изменится для пользователя:

- сообщения каждого MAX-чата приходят только в его топик;
- очереди продолжают защищать от потери сообщений при ошибках Telegram.

Что сделать технически:

- расширить callback доставки: передавать не только `chat_id`, но и `message_thread_id`;
- добавить параметр `message_thread_id` во все send-методы Telegram;
- при отсутствии топика ставить сообщения в очередь;
- после восстановления доступа или создания топика дозаливать очередь в нужную тему.

### Этап 4. Переделать интерфейс управления

Что изменится для пользователя:

- в личке виден выбранный рабочий форум;
- список MAX-чатов показывает статус топика;
- кнопки выбора отдельной группы больше нет.

Что сделать технически:

- обновить `/start` и `/chats`;
- добавить кнопку выбора/смены рабочего форума;
- заменить `admin:bind` на действия по топикам и отслеживанию;
- убрать обработку старого сценария выбора группы для конкретного MAX-чата.

### Этап 5. Переделать ответы из Telegram в MAX

Что изменится для пользователя:

- любое текстовое сообщение владельца в топике отправляется в соответствующий MAX-чат;
- сообщение владельца в Telegram остается в истории;
- reply на пересланное сообщение MAX превращается в reply внутри MAX;
- кнопка `Ответить` остается для явного ответа на конкретное сообщение.

Что сделать технически:

- находить MAX-чат по паре `forum_chat_id + message_thread_id`;
- принимать обычные текстовые сообщения владельца в топиках;
- хранить соответствие `telegram_message_id -> max_message_id` для пересланных сообщений;
- при Telegram reply определять исходный MAX `message_id`;
- убрать редактирование исходного Telegram-сообщения после ответа;
- убрать удаление сообщений владельца после отправки в MAX.

### Этап 6. Миграция и совместимость

Что изменится для пользователя:

- при обновлении бот не будет использовать старые ручные привязки;
- после выбора рабочего форума топики создадутся заново.

Что сделать технически:

- оставить чтение существующего `data/chat_bindings.json`;
- игнорировать старые `tg_chat_id` как маршрут доставки;
- сохранить MAX-чаты и очереди, если возможно;
- добавить версию хранилища и мягкую миграцию.

### Этап 7. Проверки и документация

Что изменится для пользователя:

- первичная настройка будет описана пошагово;
- ошибки прав Telegram будут понятны.

Что сделать технически:

- обновить тесты хранилища, маршрутизации, Telegram UI и reply-потока;
- обновить `.env.example` и README;
- прогнать точечные тесты;
- перед коммитом проверить staged-файлы и UTF-8 для измененных русскоязычных файлов.

## Риски

### У бота нет прав на топики

Риск:

- топики не создаются, сообщения накапливаются в очереди.

Снижение риска:

- запрашивать выбор форума через `request_chat` с админ-правами;
- дополнительно проверять права через Telegram API;
- писать владельцу понятную ошибку.

### Топик удален вручную

Риск:

- сохраненный `message_thread_id` больше не принимает сообщения.

Снижение риска:

- переводить чат в состояние ожидания топика;
- пробовать создать новый топик;
- сообщать владельцу о проблеме.

### Telegram reply без связи с MAX-сообщением

Риск:

- владелец отвечает reply на сообщение, у которого нет сохраненного MAX `message_id`.

Снижение риска:

- хранить связь для отправленных ботом сообщений;
- если связи нет, отправлять обычное сообщение или показывать понятное предупреждение.

## Зависимости

- Telegram Bot API forum topics: `createForumTopic`, `editForumTopic`, `message_thread_id`;
- Telegram chat selection через `KeyboardButtonRequestChat(chat_is_forum=True)`;
- существующий MAX WebSocket-клиент;
- локальная файловая система для JSON-хранилища.
